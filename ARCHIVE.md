# CLM / 초·분 단타 모멘텀 프로젝트 — 연구 완료 아카이브

## 최종 결론

현재의 데이터·시장·실행환경에서, 초·분 단위 모멘텀 추격 접근은 **종목 편중과
극단값(top winner)을 제거하면 일반화되는 기대값을 보여주지 못한다.** 세 개의
신호축을 실데이터로 검증했고, 비용·룩어헤드·종목편향을 정직하게 반영하면 어느
것도 견고한 우위를 남기지 못했다.

> **현재 검증한 신호·기간·실행환경에서는 일반화 가능한 엣지가 입증되지 않았으며,
> 추가 탐색의 기대가치가 개발비와 위험을 정당화하지 못한다.**

"엣지가 절대 없다"까지는 유한한 데이터로 증명할 수 없다. 그러나 더 투자하지
않기로 결정하기에는 증거가 충분하다. 이 프로젝트는 **실패가 아니라 연구 종료**다.
실주문 증액 전에 잘못된 근거(H2/H3 버그 + 종목편중)를 찾아냈고, 재현되지 않는
알파에 더 큰 돈과 시간을 넣지 않았다. **그게 이번 프로젝트의 실현된 수익이다.**

> **가장 값진 판단:** per-coin cap을 걸면 엣지가 사라진다는 것은 cap이 문제가
> 아니라, 신호가 종목을 넘어 일반화되지 않았다는 뜻이다. 이 한 문장이 종료를
> 감정이 아니라 검증 결과로 만든다.

## 검증한 신호축 & 판정

| 신호축 | 판정 (H2/H3 보정, 실데이터) |
|---|---|
| climax (body/wick/VR5/CS + 모든 필터) | 우위 없음 — 전 구간 마이너스 |
| RSI/EMA 과열 (페이퍼 CLM) | 우위 없음 — 무필터 ≥2% 추격과 동등 |
| 급등세기 ≥3%/1분 추격 | 견고성 실패 — 단일종목(BONK) 산물 |

급등세기 ≥3% 견고성 테스트 (152 이벤트):
- 기준 전체: PF 1.42 / +0.446%
- BONK 제외: PF **0.99** / −0.007% (본전)
- 상위5 제외: PF **0.72** / −0.334% (마이너스)
- 종목당 cap=3: PF **0.78** / −0.264% (마이너스)
- 기여 41종목 중 net+ 21 / net− 20 = 사실상 동전던지기

## 결정적 발견 3가지

1. **H2/H3 백테스트 버그가 부호를 뒤집었다.** sweep가 수수료를 곱셈(×0.9995)으로
   처리해 사실상 0 + 왕복 수수료 누락(H2), 봉 안에서 "고점 먼저·저점 나중"을
   가정(H3). 보정하니 CS40+VR3가 +0.124% → −0.076%로 전환. 실주문 승격 근거가
   버그 위에 있었다.

2. **모든 apparent 엣지가 단일종목 산물이다.** VANA → SOL → EGLD → XRP → BONK,
   프로젝트 내내 "한 종목이 alpha 전부"가 반복됐다. 구조적 함정.

3. **per-coin cap=3은 올바른 방어인데, cap을 걸면 엣지가 사라진다.** 즉 cap이
   문제가 아니라 신호 자체에 종목을 넘어 일반화되는 엣지가 없다는 뜻.

## 평생 규칙 (어떤 백테스트든 믿기 전에)

1. **수수료·슬리피지를 %p로 정확히 차감** (곱셈 haircut 금지).
   실제 실행환경 기준으로 %p 차감한다. 본 프로젝트의 기준 비용은
   왕복 약 0.20%p였으며(수수료 편도 5bp × 2 + 슬리피지 편도 5bp × 2),
   향후 재사용 시 당시 수수료·체결조건으로 다시 산정한다.
2. **Leave-One-Out** — 제일 큰 기여 종목(들)을 빼고도 엣지가 남는가.
3. **per-coin cap** — 종목당 이벤트 상한을 걸고도 기대값이 유지되는가.
   (+ 시간순 walk-forward, 봉내 룩어헤드 제거)

## 이 문서에서 가장 중요한 것 — 검증 순서

전략 성과표보다 중요한 유산은 **앞으로 어떤 전략을 보더라도 먼저 적용할
검증 순서**를 남겼다는 점이다:

```
비용 반영
  → 룩어헤드 제거
  → 시간순 검증 (walk-forward)
  → Leave-One-Out
  → Top winner 제거
  → 종목 집중도 제한 (per-coin cap)
  → 실데이터 재현
```

이 7단계를 통과 못 하는 "대박 신호"는 대박이 아니다. 이 순서 자체가 다음
프로젝트에서도 유효하다.

## 실행 결정 (확정)

1. 실주문 봇 종료 (`enabled=True→False` 또는 `AUTO_TRADE=0`)
2. 증액 계획(100k→300k→500k) 폐기
3. microstructure 개발 시작 금지 (비싼 재확인 도박)
4. 정확성 개선 커밋(vSfEc 6) + 얼럿 패치만 merge·배포
5. 현재 프로젝트 = 연구 완료 상태로 아카이브
6. 최소 며칠 신규 전략 코딩 중단
7. 재시작 시 초·분 단타의 연장이 아니라 별도 스윙 연구 제안서부터

## 재시작 조건 (스윙 연구 제안서의 최소 기준)

신호: 4시간봉·일봉 / 진입 판단: 1시간봉 / 보유: 수일~수주.
아래를 만족 못 하면 스윙도 개발하지 않는다:
- 거래 빈도보다 기대값·견고성 우선
- 수수료·슬리피지 포함
- 시간순 walk-forward
- 종목별 leave-one-out + top winner 제거 + per-coin 집중 제한
- 최소 수개월, 서로 다른 regime 포함
