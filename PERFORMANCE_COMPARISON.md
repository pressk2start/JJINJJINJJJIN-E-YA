## 5. MFE/MAE 추이 분석

```
MFE:  +0.09  +0.31  +0.41  +0.47  +0.41  +0.22  +0.11  +0.09
MAE:  -0.29  -0.29  -0.26  -0.26  -0.27  -0.26  -0.24  -0.23
R:R:   0.31   1.07   1.58   1.81   1.52   0.85   0.46   0.39
       ──────────────────────────────────────────────────────→ S1~S8
```

### MFE (Maximum Favorable Excursion, 최대 유리 이탈)

- **피크**: S4 +0.47% → 현재 S8 +0.09% (S1 수준으로 회귀)
- S1→S4: 5배 개선 (진입 품질 향상)
- S5→S8: 5배 악화 (진입 품질 되돌림)
- **MFE 하락 = 트레이드가 유리한 방향으로 충분히 움직이지 못함**

### MAE (Maximum Adverse Excursion, 최대 불리 이탈)

- 꾸준한 개선: -0.29% → -0.23% (SL 관리 효과)
- MAE 개선은 코드 변경(SL 1.0~1.8%, ATR 기반 동적 SL)의 직접적 성과

### 실질 R:R 비율 (MFE/|MAE|)

- S4 피크: 1.81:1 (유일하게 지속 가능한 수준)
- S8 현재: 0.39:1 (**극도로 불리**, 수익 << 손실)
- **문제**: MAE는 개선됐지만 MFE가 더 빠르게 악화 → R:R 구조 붕괴

## 7. 핵심 문제점 진단

### [문제1] MFE 붕괴 — 가장 시급한 문제

```
S4: MFE +0.47%, MAE -0.26% → 실질 R:R 1.81:1 ✅
S8: MFE +0.09%, MAE -0.23% → 실질 R:R 0.39:1 ❌
```

MFE +0.09%는 수수료(~0.1%)를 커버하지 못하는 수준.
**현재 구조에서는 승률이 50%여도 손실**이 발생.

**원인 가설**:
- 진입 타이밍 악화 (confirm 증가, 추가 검증 로직)
- 강한 모멘텀(점화) 기회 감소
- 모니터 권고 부분청산이 너무 빨리 발동 (이익을 충분히 키우지 못함)

### [문제2] confirm 모드의 역효과

confirm 비율이 높을수록 전체 승률 하락:
- S1-S4 (confirm 0~2건): 평균 승률 23%
- S5-S8 (confirm 6~10건): 평균 승률 23%... 같은데 MFE가 극적으로 다름

MFE 비교:
- S1-S4 평균 MFE: +0.32%
- S5-S8 평균 MFE: +0.21%

confirm이 MFE를 낮추는 메커니즘: 추가 확인 → 더 늦은 진입 → 모멘텀 이미 소진

### [문제3] "모니터 권고 부분청산" 과다

최근 10건 상세를 보면 패배 트레이드 중 "모니터 권고 부분청산"이 다수:
- 이는 소폭 이익 구간에서 조기 청산 → 이익을 키울 기회 상실
- 이후 방향이 맞았어도 이미 청산된 상태

### [문제4] 시간만료 손실컷 비중 과다

패배 트레이드의 대부분이 "시간만료 손실컷"으로 종료:
- 시간 내에 목표에 도달하지 못하고 소폭 손실(-0.1%~-0.5%)에서 종료
- 시간만료 = 모멘텀 부족 = 진입 타이밍이 늦었거나 신호 품질이 낮았음

## 8. 개선 권고 (우선순위)

### [R1] MFE 회복 — 최우선

현재 MFE +0.09%는 수수료보다 낮아 구조적으로 수익 불가능.

**구체적 조치**:
1. **confirm 모드 비율 축소** (10건→3건 이하) 또는 confirm 판단 시간 단축
2. **모니터 권고 부분청산 기준 완화**: 현재 너무 일찍 발동 → MFE 타겟 도달 전 청산 방지
3. **점화 경로 진입 조건 재검토**: 27건→40건+ 수준으로 복귀 필요

**목표**: MFE를 +0.30% 이상으로 회복 (R:R > 1.3:1)

### [R2] 진입 경로 포트폴리오 최적화

현재 상태:
```
점화: 27건(22%) ← 건수↓, 승률↓ (과거 42건/32%)
EMA↑: 19건(32%) ← 건수↑, 승률↑ (성장 경로)
거래량↑: 3건(33%) ← 아직 소규모
강돌파: 1건(100%) ← 극소수
```

**권고**:
- **EMA↑ 경로 확대**: 현재 가장 안정적. 필터 약간 완화하여 25건+ 목표
- **점화 경로 복구**: S3-S4 시절의 필터 수준으로 롤백 검토 (42건, 29-32%)
- **강돌파 조건 완화**: 발생 빈도 높이기 (현재 너무 엄격)
- **거래량↑ 경로 관찰 유지**: 표본 부족, 데이터 축적 후 판단

### [R3] 청산 로직 개선 — 수익 극대화

**현 문제**: 승리 트레이드의 평균 수익이 너무 작음
- "모니터 권고 부분청산"이 +0.03%~+0.10% 수준에서 발동
- 이는 수수료 감안 시 실질 무수익

**권고**:
1. 모니터 권고 부분청산의 최소 수익 기준 상향 (예: +0.3% 이상일 때만)
2. 트레일링 발동 후 최소 보유 시간 도입 (예: 30초)
3. MFE 타겟 도달률 모니터링 → 미달 시 자동 파라미터 조정

### [R4] 임밸런스 필터 강화

모든 스냅샷에서 일관:
- 승리: 임밸런스 양수(매수 우위)
- 패배: 임밸런스 음수(매도 우위)

**권고**: 임밸런스 < -0.3 일 때 진입 스킵 또는 포지션 축소

### [R5] 시간만료 손실컷 분석 및 대응

시간만료 손실컷의 평균 손실이 -0.2%~-0.5% 수준.

**권고**:
1. 시간만료 전 일정 시간(예: 60초)에 손실 -0.1% 이하면 조기 청산 (더 큰 손실 방지)
2. 시간만료 기준 자체를 단축 (빠른 실패 인정)
